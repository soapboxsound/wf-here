SUPPLEMENTAL BRIEF S1 — GOOGLE PLACES INTEGRATION

This brief adds Google Places lookup to two existing pages: the submit form and the explore page search bar. It does not rebuild either page — it layers on top of what is already built. Mapbox remains the map renderer. Google Places is used only as a data source for place search and autocomplete.

BEFORE STARTING — MANUAL SETUP

Step 1 — Get a Google Places API key. Go to console.cloud.google.com. Create a new project called "wf-here" or use an existing one. Go to APIs and Services then Enable APIs. Enable these two APIs: Places API and Maps JavaScript API (needed for the Autocomplete widget even though we are not showing a Google map). Go to APIs and Services then Credentials. Click Create Credentials then API Key. Copy the key.

Step 2 — Restrict the API key. Click the key you just created. Under Application restrictions select HTTP referrers. Add these referrers: https://wf-here.com/*, https://*.vercel.app/*, http://localhost:*. Under API restrictions select Restrict key and select Places API and Maps JavaScript API. Save.

Step 3 — Add the key to Vercel environment variables. In your Vercel project settings add a new environment variable: GOOGLE_PLACES_API_KEY with your key value. Also add it as a local environment variable for development — create a file called .env in your project root (if it does not exist) and add: GOOGLE_PLACES_API_KEY=YOUR_KEY_HERE. Add .env to your .gitignore file so the key is never committed to GitHub.

Step 4 — Create a Vercel serverless function to proxy Google Places calls. This keeps your API key server-side and out of the browser. Create a new file at /api/places.js. This is a Vercel serverless function — Vercel automatically serves any file in the /api folder as an API endpoint.

/api/places.js contents — Cursor writes this:

const fetch = require('node-fetch');

module.exports = async (req, res) => {
  const { action, input, placeId, sessionToken } = req.query;
  const API_KEY = process.env.GOOGLE_PLACES_API_KEY;

  if (!API_KEY) {
    return res.status(500).json({ error: 'API key not configured' });
  }

  if (action === 'autocomplete') {
    const url = `https://maps.googleapis.com/maps/api/place/autocomplete/json?input=${encodeURIComponent(input)}&types=establishment&key=${API_KEY}&sessiontoken=${sessionToken}`;
    const response = await fetch(url);
    const data = await response.json();
    return res.status(200).json(data);
  }

  if (action === 'details') {
    const fields = 'name,formatted_address,geometry,opening_hours,photos,types,price_level,formatted_phone_number,website';
    const url = `https://maps.googleapis.com/maps/api/place/details/json?place_id=${placeId}&fields=${fields}&key=${API_KEY}&sessiontoken=${sessionToken}`;
    const response = await fetch(url);
    const data = await response.json();
    return res.status(200).json(data);
  }

  return res.status(400).json({ error: 'Unknown action' });
};

Also run in terminal from project root: npm install node-fetch@2

This proxy means the Google API key never appears in your browser JavaScript. All Places calls go through /api/places which adds the key server-side.

Create a new file /js/places.js. This is the client-side module that calls the proxy and returns clean data. Cursor writes this file.

JAVASCRIPT FOR PLACES SEARCH IN SUBMIT FORM

In /js/places.js create and export these functions:

Function generateSessionToken — returns a random UUID string. Call this once per search session and reuse it for both autocomplete and details calls within the same session. Reset it after a details call completes.

Function autocomplete — accepts an input string and sessionToken. Calls /api/places?action=autocomplete&input=INPUT&sessionToken=TOKEN. Returns the predictions array from the response. Each prediction has a description (display text) and a place_id.

Function getPlaceDetails — accepts a placeId and sessionToken. Calls /api/places?action=details&placeId=PLACEID&sessionToken=TOKEN. Returns a clean object with these fields extracted from the Google response:

name — result.name
address — result.formatted_address
lat — result.geometry.location.lat
lng — result.geometry.location.lng
hours — result.opening_hours?.weekday_text joined with line breaks, or null
isOpenNow — result.opening_hours?.open_now boolean, or null
photoReference — first photo reference from result.photos array, or null
placeType — first item from result.types array that is not "point_of_interest" or "establishment", or "other"
phone — result.formatted_phone_number or null
website — result.website or null

Function getPhotoUrl — accepts a photoReference string. Returns the URL: /api/places?action=photo&photoReference=REFERENCE. Add a photo action to /api/places.js that proxies the Google photo URL.

Add to /api/places.js a photo action:

if (action === 'photo') {
  const { photoReference } = req.query;
  const maxWidth = 400;
  const photoUrl = `https://maps.googleapis.com/maps/api/place/photo?maxwidth=${maxWidth}&photo_reference=${photoReference}&key=${API_KEY}`;
  const response = await fetch(photoUrl);
  const buffer = await response.buffer();
  res.setHeader('Content-Type', response.headers.get('content-type'));
  res.setHeader('Cache-Control', 'public, max-age=86400');
  return res.send(buffer);
}

AUTOCOMPLETE BEHAVIOR

In the submit form script import autocomplete, getPlaceDetails, getPhotoUrl, generateSessionToken from places.js.

Generate a session token on page load: let sessionToken = generateSessionToken().

As the user types in placeSearchInput (with a 300ms debounce) call autocomplete with the input value and sessionToken. Show results in placeAutocompleteList. Hide the list if input is empty or fewer than 3 characters.

When a suggestion is clicked: hide the dropdown, call getPlaceDetails with the place_id and sessionToken, then reset sessionToken to generateSessionToken(). Populate the preview card with the returned data. Store the full place details in a variable called selectedPlace. Show the preview card.

When the Look up button is clicked and the input contains "maps.app.goo.gl" or "maps.google.com": extract the place ID from the URL if possible, or use the full URL as a search query to autocomplete. This handles users who paste a Google Maps share link instead of typing a name.
